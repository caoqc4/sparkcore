# 角色素材导入说明

## 1. 目标

这套素材库现在按两个池子管理：

- `Portrait Pool`
  角色形象图素材池
- `Audio Pool`
  角色音频样例池

创建角色时，前端会按数据库字段筛选素材，而不是直接扫文件夹。  
所以本地文件夹的作用是：

1. 统一管理原始素材
2. 通过导入脚本上传到 `character-assets` bucket
3. 自动把筛选所需的字段同步到数据库

## 4. 形象图如何标记

### 4.1 基础分类来自目录

例如：

`portrait-pool/realistic/female/aurora.webp`

脚本会自动识别：

- `style = realistic`
- `gender_presentation = female`

也就是说，按性别、风格筛选时，最基础的信息已经从目录里拿到了。

### 4.2 补充标签来自同名 `.json`

例如：

`portrait-pool/realistic/female/aurora.json`

```json
{
  "display_name": "Aurora",
  "style_tags": ["warm", "soft", "companion"],
  "default_for_character": "caria"
}
```

含义：

- `display_name`
  前台展示名
- `style_tags`
  额外筛选标签
- `default_for_character`
  把这张图标记成某个 preset 的默认主图

导入后会生成：

- `gender_presentation = female`
- `style_tags = ["realistic", "warm", "soft", "companion"]`
- `metadata.character_slug = "caria"`
- `metadata.variant = "main"`

### 4.3 推荐标签

建议形象图优先使用这些维度：

- 风格类型：`realistic`、`anime`、`illustrated`
- 气质：`warm`、`steady`、`playful`
- 使用场景：`companion`、`assistant`
- 画面感觉：`soft`、`clean`、`bright`

## 5. 音频如何标记

### 5.1 基础分类来自目录

例如：

`audio-pool/audio-elevenlabs-v3/female/eleven-v3-warm-muse.mp3`

脚本会自动识别：

- `model_slug = audio-elevenlabs-v3`
- `voice_key = eleven-v3-warm-muse`
- `gender_presentation = female`

### 5.2 补充标签来自同名 `.json`

例如：

`audio-pool/audio-elevenlabs-v3/female/eleven-v3-warm-muse.json`

```json
{
  "display_name": "Warm Muse",
  "style_tags": ["Warm", "Soft", "Calm"],
  "tier": "free",
  "sort_order": 10,
  "is_default": true,
  "default_for_character": "caria"
}
```

导入后会更新 `product_audio_voice_options`：

- `display_name`
- `gender_presentation`
- `style_tags`
- `tier`
- `sort_order`
- `is_default`
- `metadata.default_for_character`
- `metadata.sample_storage_path`
- `metadata.sample_public_url`

### 5.3 为什么音频标签建议首字母大写

当前创建角色时的音色推荐逻辑优先匹配这些标签：

- `warm`
  对应：`Warm`、`Soft`、`Calm`
- `steady`
  对应：`Calm`、`Grounded`、`Clear`
- `playful`
  对应：`Bright`、`Expressive`、`Soft`

所以为了让“按 tone 推荐音色”更准确，音频 sidecar 里的 `style_tags` 建议优先使用这组大写标签。

## 8. 脚本会自动做什么

### 8.1 形象图

脚本会：

1. 上传到 Supabase Storage bucket `character-assets`
2. 新增或更新 `product_portrait_assets`
3. 同步这些筛选字段：
   - `gender_presentation`
   - `style_tags`
   - `display_name`
   - `source_type`
   - `is_shared`
   - `is_active`
   - `metadata`

### 8.2 音频

脚本会：

1. 上传到 Supabase Storage bucket `character-assets`
2. 按 `model_slug + voice_key` 找到现有 `product_audio_voice_options`
3. 更新：
   - `display_name`
   - `gender_presentation`
   - `style_tags`
   - `tier`
   - `sort_order`
   - `is_default`
   - `metadata.sample_storage_path`
   - `metadata.sample_public_url`

---

## 9. 当前限制

- 形象图可以由脚本自动建表或更新
- 音频不会自动新建 voice option，只会更新已有记录
- 所以音频导入前，数据库里必须先存在对应的 `model_slug + voice_key`
- 当前 preset 默认主图可以通过 `default_for_character` 标记
- 当前 preset 默认音色也可以通过音频 sidecar 里的 `default_for_character` 标记
- 如果某个 preset 没有显式默认音色，系统才会 fallback 到原有推荐逻辑

---

## 10. 推荐实践

建议你把素材管理分成两步：

1. 先保证目录分类正确
   - 图像按 `style/gender`
   - 音频按 `model_slug/gender`
2. 再用 sidecar `.json` 补标签
   - 图像补气质、场景、preset 默认绑定
   - 音频补 tone 标签、层级、默认项顺序

这样后面新增素材时，不需要再改脚本逻辑，只要按目录和 sidecar 规范放文件即可。

## 12. 日常素材库维护流程

### 12.1 新增一张普通形象图

1. 放到正确的 `portrait-pool/<style>/<gender>/` 目录
2. 新建同名 `.json`
3. 写 `display_name` 和 `style_tags`
4. 跑导入脚本

### 12.2 新增一个普通音频样例

1. 放到正确的 `audio-pool/<model_slug>/<gender>/` 目录
2. 确认数据库里已经有对应的 `model_slug + voice_key`
3. 新建同名 `.json`
4. 写 `display_name`、`style_tags`、`tier`
5. 跑导入脚本

### 12.3 替换某个 preset 的默认主图或默认音色

1. 准备新的素材文件和 sidecar
2. 在 sidecar 中写 `default_for_character`
3. 运行导入脚本
4. 到创建页或 role 页确认是否已经切换到新的默认素材

### 12.4 推荐执行顺序

建议每次都按下面顺序操作：

1. 放文件
2. 写 sidecar
3. 先 dry run
4. 再正式导入
5. 最后到页面验证
